Wordpress Advice:

  1. Think in terms of reusable module/components (like all current web development)
  2. In Wordpress, everything is a "post"--a reusable block of content. In an unmodified Wordpress site, you have two types of posts: Posts and Pages.
  3. Good publishing/writing principles apply: reduce clutter, and deploy high editing/writing standards

Some Wordpress Vocabulary and Dashboard Location (in rough order of importance):

  1. Appearance >> Themes: Select look and design for site
  2. Pages: Webpages in the classic understanding of the term, with text, audio/video, and images
  3. Posts: Text, audio/video, or images that will be treated like a blog post, i.e. sent to a summary feed
  4. Appearance >> Menus: Clusters of Pages, Posts, or even external URLs that can be inserted in various parts of the site
  5. Appearance >> Widgets: Miscellaneous functionalities such as a search bar, archive roll, etc. that can be placed in sidebars, headers, and footers of sites

Some Wordpress Vocabulary and Dashboard Locations (continued):

  1. Plugins: Code to extend the functionality of a website that can be activated or deactivated
  2. Media: Library for managing image, audio, and video assets
  3. Posts >> Categories: Hierarchial and customizable terms for grouping/labeling Posts
  4. Posts >> Tags: Customizable terms for grouping/labeling Posts

(Hidden) Important Options

  1. Most Wordpress Themes allow you to set a blog roll as your home page (i.e. root URL) or a Page.
    Set this option in the Appearance >> Customize >>Static Front Page menu.
  2. Don't want Comments on a Page or Post? Unselect the "Allow Comments" box under Page/Post >> --Page/Post Title-- >> Quick Edit
  3. Exert control over Widgets, which easily clutter up a default site. Visit Appearance >>Widgets and drag and drop those you do not want.
  4. Each Plugin behaves diffently as to how to deploy, but a few key ones here (Gallery and Google Maps) can be added from Page/Post text editor
